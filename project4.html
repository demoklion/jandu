Charles University SIS Design System

Design System for the Complete Renewal of Student Information System at One of Europe's Oldest Universities

Target 1/3

How can we modernize a legacy Student Information System serving all faculties of a historic university, ensuring a modern, responsive, mobile-first user experience while fundamentally rethinking the system architecture beyond just cosmetic improvements?

My role 2/3

The gist of it

As Design Systems Lead, I spearheaded the strategic pivot from incremental UI improvements (SIS4) to a comprehensive system renewal. Leading the design system creation for Charles University's Student Information System, I helped transform one of Europe's oldest universities' most critical student-facing platform.

More details

Initial goals included mobile-first design, improved visual aesthetics, and better user experience for common student tasks like viewing study plans, results, and managing personal data.

However, during the design and development process, it became clear that cosmetic changes alone wouldn't address the fundamental architectural limitations of the legacy system. This involved creating reusable components, establishing design patterns for complex university workflows, and ensuring the system could scale across Charles University's diverse faculty ecosystem.

The work required balancing modern user experience expectations with the complex requirements of academic administration, while maintaining accessibility standards and supporting both desktop and mobile usage patterns. I collaborated with cross-functional teams including developers, administrators, and student representatives to ensure the design system served all stakeholders.

Artifacts 3/3

  • Comprehensive design system in Figma with reusable components for academic workflows, student data management, and course administration interfaces
  • SIS4 beta release featuring modern home page, responsive dashboard with email overview, study plans, results calendar, and personal data management
  • Mobile-first responsive design patterns ensuring functionality across all device types for student accessibility
  • Design system documentation and guidelines for future development phases and cross-faculty implementation
  • Strategic roadmap for phased system rollout, moving from SIS4 proof-of-concept to complete SIS renewal
  • Established feedback mechanisms and beta testing program with direct student engagement through university counseling center
